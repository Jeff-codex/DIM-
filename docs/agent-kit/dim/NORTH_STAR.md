# DIM — NORTH STAR

## 1. 프로젝트 정의

DIM은 스타트업, 서비스, 런칭, 산업 변화를 단순히 소개하지 않고
비즈니스 구조와 맥락을 해석하는 한국어 에디토리얼 인텔리전스 매거진이다.

## 2. 프로젝트 본질

이 프로젝트의 본질은 속보형 미디어가 아니다.
핵심은 해석 권위를 축적하는 매거진이 되는 것이다.

## 3. 지금의 공개 프레이밍

- `프로젝트 제출`
- `편집 및 해석`
- `콘텐츠 발행`

즉, DIM은 읽는 매체이면서 동시에 외부 프로젝트와 런칭 정보를 받아
자체 해석 체계 안에서 다시 피처로 정리하는 브랜드다.

## 4. 핵심 타겟

- 창업가
- 스타트업 운영자
- 투자자와 전략 실무자
- 제품/브랜드 런칭을 준비하는 팀
- 단순 보도보다 구조적 해석을 원하는 독자

## 5. 사용자가 원하는 것

- 빠른 소식보다 오래 남는 해석
- 지금 무엇이 바뀌는지 읽는 프레임
- 신뢰 가능한 논리와 정리된 구조
- 축적되는 아카이브형 피처
- 다시 돌아올 이유가 있는 브랜드 시선

## 6. 절대 목표

- DIM만의 해석 권위를 누적한다
- 읽을수록 더 강해지는 아카이브를 만든다
- 브랜드 검색과 주제 검색에서 DIM 피처가 독립 문서로 인식되게 만든다
- 프로젝트 제출을 권위 훼손 없이 피처 체계 안으로 흡수한다
- 장기적으로 교육/포럼/브랜드 확장으로 연결될 기반을 만든다

## 7. 현재 단계

현재 DIM은 기초 구축 단계를 넘겼다.

이미 갖춘 것:

- 공개 라우트 4+1종
- MDX 발행 구조
- 대표 이미지 체계
- review preview workflow
- SEO 기본 메타와 JSON-LD
- 매거진 문법 기반 1차 리디자인

지금부터 중요한 것은 `더 만들기`보다 `더 DIM답게 좁히기`다.

## 8. North Star Metric

- 1차: 고품질 피처 누적 수
- 2차: 검색 기반 재방문 독자 비중
- 3차: 브랜드명 직접 유입과 피처 재인용 증가
- 4차: 의미 있는 피처 제안 유입 수

## 9. 지금 하지 말아야 할 것

- 권위 형성 전의 조급한 수익화
- 얕은 트래픽 목적의 대량 발행
- 뉴스룸/제보 플랫폼 식 확장
- CMS와 운영 기능의 과한 조기 구축
- DIM 시선 없이 레퍼런스 사이트 문법만 복제하는 일

## 10. 편집 판단 원칙

1. DIM은 정보보다 해석의 질로 차별화한다
2. 모든 피처는 구조와 맥락을 함께 읽어야 한다
3. 검색 유입은 결과이고 권위 축적이 원인이다
4. 제출은 양보다 적합성과 맥락 밀도를 먼저 본다
5. 디자인과 카피는 권위를 드러내야지 과장해서는 안 된다
